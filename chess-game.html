Chess Game

Role: Solo developer · Tech: JavaScript, HTML5 Canvas

A web-based chess prototype focused on approachable UI and experimenting with AI heuristics that balance performance and quality of moves for casual play.

Problem & Objectives

The goal was to create a lightweight chess client that runs entirely in the browser, providing a responsive UI and an AI opponent with configurable depth and heuristics. Objectives: fast move generation, clear board visuals, and a friendly UX for new players.

Technical Implementation

  • Engine: Minimax search with alpha-beta pruning implemented in JS for client-side play.
  • Rendering: HTML5 Canvas for efficient board redraws and animated piece movement.
  • State: Immutable move history to enable undo/redo and simple replay.

Challenges & Solutions

Performance at deeper search depths in JS—addressed by iterative deepening and move-order heuristics. Another challenge was designing a clear, non-intimidating UI; solved by simplifying legal move highlights and offering a practice mode that suggests moves.

Key Code Snippet

// Simple alpha-beta outline (illustrative)
function alphaBeta(node, depth, alpha, beta, maximizingPlayer) {
  if (depth === 0 || node.isTerminal()) return evaluate(node);
  let moves = node.generateMoves();
  for (let m of moves) {
    node.apply(m);
    let score = -alphaBeta(node, depth-1, -beta, -alpha, !maximizingPlayer);
    node.undo(m);
    if (score >= beta) return beta;
    if (score > alpha) alpha = score;
  }
  return alpha;
}

Results & Impact

Usable, compact web demo that runs comfortably at shallow search depths on mobile and desktop. Good baseline for later server-side engine integration.

Future Enhancements

  • Connect to a stronger engine server-side (Stockfish) for advanced play.
  • Save/load PGN; add puzzles and training modes.
