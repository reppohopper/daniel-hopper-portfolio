Unity FPS Game

Technology: Built in Unity/C#

My role: Structured solo project for LWTech game programming course

Description: Small prototype focused on achieving a complete level a playable game loop. Features enemies, obstacles, HP and ammo recovery items, and win and lose conditions.

Problem & Objectives

The gaol for this assignment was to expand past the basic assignment requirements. Our professor explicitly said, "put work in it to make it your own, and try to make it interesting". I worked toward a well-rounded game with extra visual features, carefully calibrated challenges and rewards, and that was intuitive and fun to play.

Technical Implementation

  • Unity Engine GUI for 3D objcet/asset design, C# MonoBehavior scripts for player/enemy behavior and game logic.
  • Direct input management: WASD movement controls, mouse to look up, down, or turn. Click to shoot.
  • Player controller with configurable elements like movement speed, fire rate, and projectile speed.
  • Enemy controllers with variable health, sight/shoot distance, agro distance, and defined patrolling behavior.
  • Level flow including distinct sections of different atmostphere and increasing difficulty.
  • HP and ammo recovery items drop on enemy defeat.
  • Interaction of these elements are calibrated to make an interesting, moderately challenging game.
  • Intuitive "Game over" messaging with easy restart.

Key Functionality Code Snippet

The following code is just a small section of the Player Controller script, handling the response to the player taking damage and demonstrating the modular, method-based nature of the script.

 public void PlayerTakesDamage(int damage) 
      {
          // Reduce player health
          playerHealth = Math.Max(playerHealth - damage, 0); // Can't fall below zero. 
          
          // Update display health. 
          healthText.text = "HP: " + playerHealth.ToString() + " / " + playerMaxHealth.ToString();
  
          // Check to see if we are dead
          if (playerHealth <= 0) {
              gameOverText.SetActive(true);
              restartText.SetActive(true);
              SimulateFreezingTheGame();
          }
      }

      // Method to simulate Freezing the Game 
      public void SimulateFreezingTheGame() 
      {
              rotationSensitivity = 0f;
              verticalRotationSensitivity = 0f;
              ammoCount = 0; // Falsely remove all ammo to prevent final shot. 
              Time.timeScale = 0f;
      }

Challenges Faced & Solutions

The challenge here was to balance my excitement about the project's potential with the reality of the project being due on a deadline, and being just one of many competing academic and persona priorities. It was a great learning experience as I discovered efficient means of for threading this divide.

I focused on maximum code modularity and reusability, separating "looking", "moving", and "attack" functionalities so I could re-use them piecemeal for the player controller, the enemies, and the stationary turrets. Similarly, assets like the turret towers and cover obstacles made use of mirroring for quick duplication and re-use while looking rough and natural.

As a result of focusing on features I could find a way to ship quickly, I shipped a game that stood out as excellent in the class final presentation.

Potential Next Enhancements

The things I most wanted to do next to elevate the gameplay were...

  • Slight enemy strafing if they take damage or are attacking: "bob and weave".
  • Red screen flash to indicate player taking damage.
  • Sounds for damage, firing, item collection game over, and win.
